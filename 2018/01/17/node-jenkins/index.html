node-jenkins

作者:super J726    发布于: 2018-01-17

node项目在jenkins中实现自动化打包及部署。

在之前的发版项目过程中，出现几次例如，本地前端代码中环境切换未切换对应的服务器域名的错误等，因此需要更好的维护前端项目。

首先前端项目需跟后端项目一样，启用多分支，对应分支分开打包部署。

因此用jenkins来实现前端项目的自动化打包及部署。

1.新建项目

2.输入你需要构建的项目名称，例如 One_sp_b2b-front，选择自由风格。

3.参数化构建配置

3.1在构建前端项目前的JOB中增加构建后操作，当前是在One_sp_b2b-web中配置

3.2回到 One_sp_b2b-front,配置参数化构建过程，增加 string parameter

4.源码管理的配置

4.1这里的Branch Specifier用上面配置的参数

5.构建的配置

5.1. 增加构建步骤

5.2.编写node打包命令脚本（服务器需要安装Node.js)

npm install
npm run build

5.3.平时打包前端代码是打包到templates文件夹中，因此web项目中的前端代码中需要修改默认打包地址，在项目中，配置npm run build打包地址的index.js在config文件夹中,将打包地址改为../templates,即在服务器打包时，在/home/jenkins/workspace/One_sp_b2b-front/文件夹中创建一个templates文件夹

5.4.服务器执行完打包后，执行转移代码脚本，将编译后的前端代码cp到web项目中

cp -rf /home/jenkins/workspace/One_sp_b2b-front/templates/* /home/jenkins/workspace/One_sp_b2b-web/src/main/resources/templates/

5.5.继续新增一个脚本，invoke top-level Maven targets

web构建项目的脚本配置，搬过来执行一次，目的是在前端代码打完完并且cp到web项目中后，让web项目重启一遍。

clean install  -Dmaven.test.skip=true -U -Ptest

5.6.继续新增一个脚本，执行web JOB的打包构建脚本

6.完成

点apply并保存，返回面板，立即构建
